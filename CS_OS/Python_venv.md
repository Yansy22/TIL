# Python 가상환경(venv)의 개념과 구조 이해

## 1. 가상환경(Virtual Environment)이란?
- 프로젝트별로 독립된 파이썬 실행 환경을 제공하는 '격리된 공간'이다.
- 프로젝트마다 필요한 라이브러리 버전이 다를 때 발생할 수 있는 충돌을 방지한다.
- 시스템 전체(Global) 파이썬 환경을 깨끗하게 유지할 수 있다.

## 2. 가상환경의 물리적 구조
가상환경을 생성하면 프로젝트 폴더 내에 `.venv` 폴더가 생성되며 내부 구조는 다음과 같다.
- **`Include/`**: C언어로 작성된 라이브러리 컴파일 시 필요한 헤더 파일 저장소.
- **`Lib/site-packages/`**: `pip install`로 설치한 라이브러리들이 실제로 저장되는 핵심 창고.
- **`Scripts/`**: 환경을 제어하는 실행 파일들이 모여 있는 곳.
  - `python.exe`: 이 환경 전용 파이썬 엔진.
  - `Activate.ps1`: 가상환경을 활성화하는 스위치.
  - `pip.exe`: 이 환경 전용 패키지 설치 도구.

## 3. 핵심 요약
- 가상환경은 단순히 폴더 형태의 '격리된 가방'일 뿐이며, 코드 파일과는 물리적으로 분리되어 있다.
- 따라서 하나의 코드 파일에 여러 가상환경을 갈아끼우며 테스트하는 것이 가능하다.
